1. LLVM FrontEnd :

  • lexical-syntax-semantic, output AST

  • AST → CodeGen → IR (.ll)


2. MiddleEnd :

  • .ll → Optimization → .ll


3. Useful commands :

http://clang.llvm.org/get_started.html


clang -E

preprocess only

clang -fsyntax-only

check for correctness, type checking

clang -Xclang -ast-dump -fsyntax-only

dump the AST

clang -cc1 -ast-print

clang -S -Xclang -ast-print file.c

-Pretty printing from the AST to screen

-Output to file.s (not assembly but C or C++ code)

clang -S -emit-llvm file.c

Output unoptimized IR in text format (file.ll)

    clang -S -emit-llvm -O3

Optimized IR

clang -S -O3 file.c

Output native machine code (file.s assembly code)

llc foo.ll

To compile LLVM IR further to assembly, use the llc tool



-cc1 Compiler front-end, and not the driver, should be run

The 'clang' driver is designed to work as closely to GCC as possible to maximize portability


-S Only run preprocess and compilation steps (noLinking)

-c Only run preprocess, compile, and assemble steps

-Xclang <arg> Pass <arg> to the clang compiler
